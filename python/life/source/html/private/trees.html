Module Hierarchy

  • lexicon: Classes for reading Life patterns from files.
  • life: Classes for operating the graphical user interface for Conway's Game Of Life.
  • lifegrid: Classes for representing a Life grid.
  • wxwrap: Contains class for easier creation of wxPython forms.

Class Hierarchy

  • __builtin__.object: The most base type
    • __builtin__.type: type(object) -> the object's type type(name, bases, dict) -> a new type
    • threading._Verbose
      • threading.Thread
        • life.Abortable: Abortable thread.
          • life.Worker: Thread that iterates the lifegrid.LifeGrid one step forward, then posts a ResultEvent back to wxPython using wx.PostEvent.
  • lexicon.Lexicon: Class for storing Life patterns divided in classes.
  • lexicon.Patterns: Class for storing a collection of Life patterns.
  • life.LifeGUI: Class for operating Conways' Game of Life.
  • lifegrid.LifeGrid: Base class for representing a life grid.
    • lifegrid.SimpleLifeGrid: Life grid that is readable and correct but not excessively fast.
    • lifegrid.UnrolledLifeGrid: Somewhat faster implementation of life grid.
  • wxwrap.WxWrap: Class for easier creation of wxPython forms.
